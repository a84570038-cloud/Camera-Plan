# 2D 布光引擎数学原理与可行性技术验证

## 1. 2D 二维布光模拟器几何数学原理

### 1.1 坐标轴映射关系 (Screen to SVG Viewport Mapping)
画布的实际渲染基于底层的 `<svg>` 矩形容器，其内部逻辑分辨率规划为标准 `300 x 300` 虚拟网格（Collimator Grid）。当多点触控与鼠标在画布滑动时，若要定位其在 SVG 里的相对坐标 `(svgX, svgY)`，需要抵消外围各种弹性弹性容器（Flex Grid & Fluid Width Layout）的大小波动。

其坐标计算核心算式定义为：
$$svgX = \frac{clientX - rect.left}{rect.width} \times 300$$
$$svgY = \frac{clientY - rect.top}{rect.height} \times 300$$

该机制的运行完全摆脱了对 `window.innerWidth` 的固定硬编码数值依赖，支持在任何 responsive 画幅、双侧分栏或悬浮面板缩放状态下，计算出零偏差的精准像素定位点。

---

### 1.2 拖曳点智能向心对焦解析 (Smart Heading Angle Back-Prop)
在 v1.1 优化版中，引入了自动对焦角度演算：
当拍摄光源（比如软光箱或聚光灯）被从坐标 $P_1(x_1, y_1)$ 移动到 $P_2(x_2, y_2)$ 时，若要指向中央固定模特原点 $M(x_m, y_m) = (150, 150)$，其在平面直角坐标系中的所需偏向航向角 $\theta$ 的三角函数解法为：

1.  **偏差分量**:
    $$\Delta x = x_m - x_2 = 150 - x_2$$
    $$\Delta y = y_m - y_2 = 150 - y_2$$

2.  **反正切正弧度**:
    $$\text{radian} = \text{atan2}(\Delta y, \Delta x)$$

3.  **角度转换并规整**:
    $$\theta_{\text{deg}} = \text{radian} \times \frac{180}{\pi}$$
    $$\text{if } \theta_{\text{deg}} < 0 \implies \theta_{\text{deg}} = \theta_{\text{deg}} + 360$$

通过这一算式，在 `handlePointerMove` 时计算出的向心弧角将实时回填到 `device.angle` 里，无需创作者在拖拽后费力调节角度面板。

### 1.3 光束散射羽化扇形 (Beam Sector & Linear Gradient Geometry)
对于 Strobe、Softbox 或 Spotlight，其投射出去的光斑不仅仅是一根虚线，而是一个具有特定发散夹角 $2\alpha$ 的发散锥（cone）。在 SVG 中，这个发散锥是用包含弧线连接的三点路径扇形来表达：

已知光源位置为 $P(x, y)$，主指向角为 $\theta$，发散半角为 $\alpha$（默认 $\alpha = 22^\circ$），光轨延伸距离为 $L = 180$。其出射光圈切面两端极值拐点 $Q_1(x_{q1}, y_{q1})$ 和 $Q_2(x_{q2}, y_{q2})$ 为：
$$x_{q1} = x + L \cos(\theta - \alpha)$$
$$y_{q1} = y + L \sin(\theta - \alpha)$$
$$x_{q2} = x + L \cos(\theta + \alpha)$$
$$y_{q2} = y + L \sin(\theta + \alpha)$$

结合 SVG 的 `Path` 绘制指令：
`d="M {x} {y} L {x_q1} {y_q1} A {L} {L} 0 0 1 {x_q2} {y_q2} Z"`
此路径能产生一个光影三角扇形切面。外加与之相重合绑定的 `radialGradient` 径向彩温渐变，从极值光束中心向四周发爆散扩散，生动精确地模拟出摄影柔光在空气中的高仿真衰减特效。

## 2. AI 策划大纲服务：结构化 JSON Schema 验证 (Structured Outputs)
本项目的后端 API (`/api/generate-plan`) 使用最新版 `@google/genai` TypeScript 开发套件。通过强约束的 `ResponseSchema` 策略，强迫 AI 大模型生成符合严格 TypeScript 定义的 JSON。

```typescript
const responseSchema = {
  type: Type.OBJECT,
  properties: {
    title: { type: Type.STRING },
    concept: { type: Type.STRING },
    wardrobeGuide: { type: Type.STRING },
    propsGuide: { type: Type.STRING },
    locationGuide: { type: Type.STRING },
    lightingInstructions: { type: Type.STRING },
    recommendedDevices: {
      type: Type.ARRAY,
      items: {
        type: Type.OBJECT,
        properties: {
          type: { type: Type.STRING },
          x: { type: Type.INTEGER },
          y: { type: Type.INTEGER },
          angle: { type: Type.INTEGER },
          power: { type: Type.INTEGER },
          colorTemp: { type: Type.INTEGER },
          label: { type: Type.STRING }
        },
        required: ["type", "x", "y", "angle", "power", "colorTemp", "label"]
      }
    }
  },
  required: ["title", "concept", "wardrobeGuide", "propsGuide", "locationGuide", "lightingInstructions", "recommendedDevices"]
};
```
该 schema 防御机制确保了即便在大模型极速响应、高负荷运转的情况下，亦绝对不会出现 JSON 解析报错。
